Build the Automation

01
Build Your Telegram Bot
1
In Telegram, search @BotFather and open the chat. Tap Start.
2
Send /newbot. Give it a display name (e.g. Signal Loop) and a username ending in _bot.
3
BotFather replies with an HTTP API token. Copy it and keep it private — this is your bot key.
4
Send your new bot any message so it's allowed to message you back.
5
Get your Chat ID by messaging @userinfobot — it replies with your numeric ID. Save both the token and Chat ID.
02
Set Up the Google Sheet
1
Create a new Google Sheet named Signal Loop Log.
2
Paste these headers into row 1: Timestamp · Source · Relevant · Why Relevant · Summary · Meta Actions · Google Actions · AEO Actions · Priority Action
3
The last two columns are yours to fill manually — this is how you track which inputs actually moved your numbers.
03
Wire the Make Scenario
1
Create a free account at make.com. Connect Gmail, Google Sheets, and Telegram.
2
In Gmail, create a label called SignalLoop and set a filter so all your newsletter senders automatically get that label.
3
Module 1 — Gmail: Watch Emails. Set folder/label to SignalLoop. Schedule: every 15 minutes.
4
Module 2 — Anthropic Claude: Create a Message. Model: claude-sonnet-4-6. Max tokens: 1500. Paste the Signal Loop prompt (Step 04) and insert the email body where marked. Add your API key from console.anthropic.com.
5
Module 3 — Parse JSON. Feed it Claude's output text. Run once to generate the data structure so fields map cleanly.
6
Module 4 — Google Sheets: Add a Row. Map each parsed field to its column. For action lists, join the array with a newline.
7
Module 5 — Telegram Bot: Send a Message. Connect using your bot token and numeric Chat ID. Set parse mode to HTML for bold formatting.
8
Run once to test. When the brief lands in Telegram and the row lands in Sheets, switch scheduling ON.
04
Drop in the Claude Brain
!
Edit the ACCOUNT CONTEXT block below to match your actual business. This one block is the difference between a real decision tool and a toy.
// Signal Loop Prompt — Paste into Make Module 2
You are the Signal Loop analyst. You read paid-media newsletters and decide what a specific operator needs to act on.

ACCOUNT CONTEXT (edit this block to match your account)
- Channels I run: Meta Ads (Advantage+ Shopping), Google Ads (PMax + Search), AEO
- Meta priority: scaling creative volume, reducing CPAs
- Google priority: feed quality for PMax, Smart Bidding signals
- AEO priority: getting cited in AI Overviews and ChatGPT answers
- I do NOT care about: TikTok, affiliate, Amazon, email marketing

TASK
Read the newsletter below. Return ONLY valid JSON — no preamble:

{
  "source": "newsletter name and author",
  "relevant": "Yes or No",
  "why_relevant": "one line — if No, name why it was skipped",
  "summary": ["3 to 5 short bullets, plain language"],
  "meta_ads_actions": ["specific actions for MY Meta account"],
  "google_ads_actions": ["specific actions for MY Google account"],
  "aeo_actions": ["specific actions for MY AEO setup"],
  "priority_action_this_week": "the single most important action to take"
}

RULES
- If a section has no genuine implication for my channels, return an empty array []
- Tie every action to something in the newsletter — no generic advice
- Keep each action under 20 words

NEWSLETTER
{{ paste the email body here from Make Module 1 }}
