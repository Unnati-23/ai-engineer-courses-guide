How to use this guide

Pick the track that matches where you want to end up — not where you are right now. If you've never written code and want to stay non-technical, start with AI Ethics & Governance. If you're ready to learn to build, AI Engineer is your foundation. If you already have some technical comfort and want a client-facing, high-impact role, Forward Deployed Engineer is the path.

Within each phase, pick ONE resource that matches your learning style — don't try to do everything. Depth beats breadth.

Track 2 · Technical

Forward Deployed Engineer

For people who want to combine technical AI skills with client-facing work — implementing AI solutions directly inside organizations.

Given a strong software engineering background, the biggest deltas for this role are usually: LLM-native engineering patterns, agent infrastructure (MCP, skills, sub-agents), evaluation discipline, and the consulting half of the job — discovery, scoping under ambiguity, and translating between executives and engineers. Ordered roughly foundations → advanced → role-shaped soft skills.

1

Foundations of LLM-native engineering

The mental shift from "API that returns text" to "stochastic component you have to engineer around." Read broadly across labs — not just one.

Anthropic — Building Effective AI Agents Essay

The canonical essay distinguishing workflows (predefined code paths) from agents (LLMs dynamically directing tool use). Covers prompt chaining, routing, parallelization, orchestrator-workers, evaluator-optimizer.

Anthropic — Effective Context Engineering for AI Agents Essay

The follow-up, defining "context engineering" as the next phase after prompt engineering.

OpenAI — A Practical Guide to Building Agents PDF

OpenAI's own framing of the same shift — useful to read alongside Anthropic's essays for a vendor-agnostic picture.

Chip Huyen — AI Engineering (book) Book

The most comprehensive practitioner text. Covers eval, RAG, fine-tuning tradeoffs, and inference optimization end-to-end.

Practitioner blogs Ongoing

Simon Willison (pragmatic, current, vendor-agnostic), Hamel Husain (especially the eval posts), and Eugene Yan (applied ML/LLM patterns from someone who's shipped at scale) — worth following on an ongoing basis.

2

The major API surfaces

Be fluent in at least one frontier API end-to-end, and literate in the others — customers always ask "why not the other guy."

Anthropic Claude docs Docs
OpenAI API docs Docs
Google Gemini API docs Docs
Meta Llama (open weights) Docs
Core concepts to internalize regardless of provider Checklist

The chat completions / messages loop and streaming; tool use / function calling (schema design, the tool_call → tool_result loop, parallel tool calls); structured outputs (JSON mode, schemas); prompt caching (a huge cost/latency lever in production); batch APIs (often 50% cost for non-realtime workloads); reasoning / extended-thinking models and when they're worth the latency; context windows, attention budgets, and how performance degrades at the tail.

Deliberate sub-project Build

Ship one real app on the API of your target lab, instrumented with caching + tool use + a proper eval harness. Forces depth.

Agent infrastructure & protocols

This is where most enterprise FDE work lives today. The patterns are converging across labs.

CrewAI, AutoGen, Mastra Know they exist

Concrete build to do once Build

5

Retrieval & RAG — without over-indexing

What to actually understand Checklist

Chunking strategies, hybrid search (BM25 + dense), reranking, evaluation (recall@k, MRR), and when to use a graph/structured approach instead. Don't spend weeks building a vector DB from scratch — understand at the level of "when to use it, how to evaluate it."

8

Enterprise vertical context

The JD almost always calls out "financial services, healthcare/life sciences, or another enterprise vertical" as a plus. Pick one and go deep enough to hold a 30-minute discovery conversation with a domain expert.

Financial services Vertical

SR 11-7 (model risk management), trade lifecycle basics, KYC/AML workflows, how compliance reviews AI systems. Reference customers: Bridgewater, Brex, JPMorgan's AI work.

Healthcare / life sciences Vertical

HIPAA, PHI handling, FDA's framework for AI/ML-based SaMD, clinical documentation workflows, EHR integration patterns.

Legal Vertical

Contract review workflows, document retention, privilege. Reference companies: Harvey, Hebbia, Ironclad.

Customer service / CX Vertical

Ticketing, deflection, agent assist, voice. Reference companies: Intercom (Fin), Sierra, Decagon.

Reading Tip

Each frontier lab publishes customer stories — read 5-10 across verticals to see what real deployments look like.

Practical exercise Do this

Take one of your past complex projects and write it up as a case study — situation, customer/stakeholder goal, constraints, what you built, measured outcome. FDE interview loops lean heavily on these.

10

Building your evidence portfolio

Concrete artifacts an interviewer can read in 10 minutes carry more weight than anything else. Aim for 3-4 of these:

  • An MCP server for a non-trivial enterprise-shaped API. Open-source on GitHub with a clear README and a 60-second demo video.
  • A custom Agent Skill (or equivalent for whichever lab you're targeting) that does something you actually use.
  • A small agent that solves a real workflow you do every week. Show eval results, not just the demo.
  • A teardown post: pick one of the frontier engineering posts from Stage 1, implement the pattern, write 800-1200 words on what worked and what didn't. This is exactly the kind of artifact FDEs produce internally.
  • A "deployment pattern" writeup: pick a customer profile (e.g. "mid-size insurance company doing claims triage"), sketch the architecture (which cloud, RAG approach, eval setup, guardrails). One page.
Suggested 8-Week Sequence

From Foundations to Portfolio

Week 1

Rebuild one piece of an existing app of yours on a frontier API with tool use + prompt caching.

Week 2

MCP. Ship one MCP server publicly.

Week 3

Agent Skills (or your target lab's equivalent). Read 3 reference skills, ship one custom skill that composes with your MCP server.

Week 4

Evals. Pick one framework, add an eval harness to the Week 1 project, get a baseline number.

Week 5

Multi-agent + sub-agents. Build one orchestrator + 2 sub-agents pattern.

Week 6

Deployment surfaces. Deploy your agent to Bedrock or Vertex. Document the security model and data flow.

Week 7

Vertical depth. Pick one vertical from Stage 8, build a domain-shaped demo on top of your existing infrastructure.

Week 8

Polish. Write up 2-3 artifacts as case studies. Practice talking through them out loud — discovery questions, tradeoffs, what you'd do differently.

Things explicitly worth not over-investing in

Building a vector DB from scratch.

The job is about deploying LLMs well, not winning a RAG benchmark.

Fine-tuning.

Most frontier labs' top models aren't user-fine-tunable, and the FDE motion almost never needs it. Knowing why prompting + context engineering replace most fine-tuning use cases is more valuable than fine-tuning skill itself.

Framework depth for its own sake.

Know LangChain / LlamaIndex / CrewAI exist; customers use them. Don't build your reference implementations on top of heavyweight frameworks — most senior engineers in this space prefer thin, direct API calls.

Chasing every new model release.

Frontier moves weekly. Build for capability classes (reasoning / tool-using / cheap-and-fast / multimodal), not specific model names.

Track 3 · Non-Tech

AI Ethics & Governance

For people from any background — policy, law, HR, business, social science — who want to work on the rules, frameworks, and oversight that govern how AI gets used. No coding required.

1

Build your foundation

Understand what AI is, how organizations think about it, and where ethics fits in.

Elements of AI Free · Certificate

Start here if you haven't already (see Stage 0 of the main guide) — covers what AI can and can't do, and its societal impact, with no math or code.

AI For Everyone — Andrew Ng Free (Audit)

DeepLearning.AI, ~6 hours. Built for non-technical people — how organizations adopt AI, including the ethical and strategic considerations.

AI Governance and Ethics — Alison Free · Certificate

Fully free, including the certificate (80% pass mark). Covers the laws, frameworks, and procedures that govern AI, and key ethical challenges.

2

Learn the frameworks that matter

These are the actual standards and regulations companies build their governance programs around.

AI Governance — University of Oxford (Saïd Business School) Free (Audit)

Covers why AI systems fail, key risks (bias, misalignment, overreliance), and how to translate ethical principles into practical oversight.

Generative AI: Governance, Policy, and Emerging Regulation — University of Michigan Free (Audit)

Covers data management, transparency, risk assessments, and the current regulatory landscape across the US, EU, and G7 countries.

NIST AI Risk Management Framework Free · Primary source

The actual framework many organizations use to structure AI governance. Free, public, and directly citable — reading the source itself is a strong signal in interviews.

3

Apply it — build your case study portfolio

For governance roles, employers want evidence you can analyze real situations, not just recite frameworks.

Pick real incidents and write your own analysis Hands-on

Choose 3-4 real, publicly reported AI governance issues (biased hiring tools, content moderation failures, deepfake incidents) and write a short analysis: what went wrong, which framework from Stage 2 would have caught it, and what you'd recommend. This becomes your portfolio.

4

Stay current & build community

AI governance moves fast — regulations and frameworks change. Staying current is part of the job.

Montreal AI Ethics Institute — The AI Ethics Brief Free newsletter

A widely-read, free weekly roundup of AI ethics and governance news — subscribe and start referencing current events in your case studies.
