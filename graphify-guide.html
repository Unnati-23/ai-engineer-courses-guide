Stop letting Claude Code re-learn your codebase every session

A walkthrough of Graphify — the open-source skill that turns your project into a knowledge graph so Claude can navigate it instead of re-reading it.

The problem with every new session

Claude Code doesn't carry a map of your project from one session to the next. Each new session starts fresh — so when you ask it to do something on a large codebase, it has to explore: searching, grepping, and reading files just to figure out how everything connects, before it even gets to your actual request.

On a small project, that's barely noticeable. On a project with hundreds of files, that orientation cost adds up fast — and it's tokens spent before any real work happens.

What Graphify actually does

Graphify is a free, open-source skill (MIT licensed) that reads your project once and builds a knowledge graph — every file, function, and how they connect — using local static analysis (tree-sitter). No code leaves your machine for this part.

Without it

Claude greps through files one by one to figure out "where does X connect to Y?"

With it

Claude reads the graph first — sees the structure, jumps straight to the relevant files.

Set it up (4 steps)

1

Install the package

Recommended via uv — it puts the command on your PATH automatically.

uv tool install graphifyy
2

Register the skill with Claude Code

This makes the /graphify command available inside your sessions.

graphify install
3

Build the graph

Run this once inside your project root. It outputs an interactive graph, a plain-language report, and a queryable JSON file.

/graphify .
4

Make Claude always check it first

This sets up a hook so Claude consults the graph before grepping raw files in future sessions.

graphify claude install

Before you install — the honest version

  • The "70x fewer tokens" number is real, but it's the best case. It applies to large codebases (500+ files). On smaller projects (under ~30 files), Graphify's own docs say you won't notice much difference — Claude can already read everything cheaply.
  • It doesn't change your subscription tier. A $20 plan stays a $20 plan — what changes is how many tokens get "wasted" on orientation versus spent on your actual question. The savings compound the most on big, ongoing projects.
  • Code analysis is local and free. Tree-sitter parses your code on-device with zero API calls. Only the optional "deep" semantic mode (for docs, PDFs, images) uses an LLM.
  • Best for: large, ongoing codebases where Claude keeps re-exploring the same structure. Skip it for: small scripts or one-off projects — the setup overhead won't pay off.
